Understand the Concept of an LLM

Imagine a small town hall — like the one in Trifouillis-sur-Loire — where Josiane, the front-desk agent, is overwhelmed with daily questions:

  • What documents are needed to get married?

  • How do you apply for a grant for an energy-efficient renovation?

  • When will the road construction on the D42 be finished?

With bulky binders and unreliable spreadsheets, Josiane spends precious time hunting for answers. The challenge is universal: how do you access accurate information quickly?

That's where you come in, as a data engineering consultant. Brought in to modernize this small town hall, you have a clear plan: to lighten Josiane's workload, you propose an innovative solution built on two pillars:

  • A vector database: A digital library that stores local information as mathematical vectors, enabling fast and accurate search.

  • A Large Language Model (LLM): An intelligent model that interprets questions in natural language and delivers tailored responses — just like a skilled human assistant.

What is an LLM?

An LLM is a program designed to understand and generate human text. Think of a student who has read millions of books and articles. Thanks to all that learning, they can answer questions or write text with impressive accuracy.

But I'd rather show you a quick usage example:

from transformers import AutoModelForCausalLM, AutoTokenizer\nimport torch\n\n# Determine the device to use (GPU if available)\ndevice = torch.device("cuda" if torch.cuda.is_available() else "cpu")\n\n# Load the model\nmodel_name = "Qwen/Qwen2.5-1.5B"\ntokenizer = AutoTokenizer.from_pretrained(model_name)\nmodel = AutoModelForCausalLM.from_pretrained(model_name).to(device)\n\n# Preprocess the user input\nquestion = "What is the capital of France?"\ninputs = tokenizer(question, return_tensors="pt", padding=True, truncation=True)\ninputs = {key: value.to(device) for key, value in inputs.items()}\n\n# Model prediction\nwith torch.no_grad():\n    output = model.generate(inputs["input_ids"],\n                            attention_mask=inputs["attention_mask"],\n                            max_new_tokens=50,\n                            num_return_sequences=1)\n    response = tokenizer.decode(output[0], skip_special_tokens=True)\n\nprint("Question:", question)\nprint("Answer:", response)

Let's look at what's happening inside what we can currently call a \"black box.\"

Imagine an invisible assistant that can instantly answer any question — no binders to search through, no calls to make. This assistant already "knows" everything it needs to know. That's exactly what LLMs do — large language models. Built to understand and generate text with impressive accuracy, these models use billions of parameters to process complex information. An LLM works like a language expert: when it reads a sentence, it anticipates each next word and assembles coherent, relevant responses.

The Core of an LLM: Parameters and Data

LLMs can answer a wide variety of questions thanks to two fundamental elements: parameters and data.

  • Parameters act like adjustable dials that modulate the model's capabilities. The more parameters an LLM has, the better it can interpret nuances and complex relationships between words. A model with few parameters might simply state that \"Paris is a city in France,\" while a more sophisticated model could also incorporate information about Paris's history, culture, or geography.

  • Data plays an equally crucial role. LLMs are trained on massive amounts of text from a wide range of sources: academic papers, books, online discussions, and more. Through this training, the model learns to recognize the relationships and meanings behind words — as if it had read every encyclopedia, plus personal letters and cooking recipes.

What exactly happens when I ask an LLM a question?

Here's what happens:

  • Decoding the question
    The model starts by \"reading\" the question sequentially — a bit like a person following a recipe. It breaks the sentence down into elementary units (called tokens) and identifies the links between them. For example, connecting \"documents\" and \"marriage\" helps place the context within an administrative domain. This step allows the model to determine what type of response is expected (a list, an explanation, etc.).

  • Activating statistical memory
    Rather than querying a database, the LLM draws on statistical memory built up during training on vast amounts of text. For a question like \"What documents do I need to get married?\", it activates familiar patterns: a government-issued ID is typically required, a birth certificate is associated with civil registration, and so on.

  • Incremental response generation
    The response is built word by word, progressively, with the model anticipating the most likely continuation at each step.

  • First step: The model starts by generating a prompt, for example \"You will need a…\" At this point, it evaluates several possible continuations (such as \"government-issued ID\", \"photo\", or \"authorization\").

  • Next step: From the completed prompt (\"You will need a government-issued ID and a…\"), it predicts the most contextually coherent word (for example \"proof of address\", \"certificate\", etc.).
    This process repeats until the response is complete or a technical limit is reached.

  • Real-time filtering and adjustment
    In parallel, filtering mechanisms step in to prevent the generation of incoherent or inappropriate responses. For example, if the probability of generating an off-topic continuation (like \"a bouquet of flowers\") is very low, that option will be excluded.

Think of a massive autocomplete system, trained on every library in the world. The LLM doesn't actually \"think\" — it assembles linguistic patterns statistically, like a chef combining familiar ingredients to create a new dish. These linguistic units are called tokens — let's explore what they are!

Understand Why an LLM Uses Tokens

Language models use tokens because they don't process text the way humans do. Unlike us, who understand sentences holistically, an LLM must convert language into a numerical representation it can work with. Tokenization enables this conversion by splitting text into smaller units (tokens), which are then transformed into numbers and processed by the model.

This approach offers several advantages:

  • Rather than storing every word in a language, LLMs use a limited set of tokens, allowing them to handle multiple languages and unknown words.

  • Working with tokens improves model efficiency by limiting the amount of information to process.

  • By splitting words into sub-units (like \"encyclo\" and \"pedia\"), an LLM can recognize and understand words it has never seen before.

Understand How an LLM Uses Tokens

Several steps take place when we run an LLM:

  1. When text is submitted to the model, it is first split into tokens. A token can be a full word, part of a word, or even a single character, depending on the tokenization method used.

  2. Each token is mapped to a unique identifier in a predefined vocabulary. For example, the word \"cat\" might correspond to the number 3456.

  3. These sequences of numbers are then analyzed by the LLM, which detects relationships between tokens using mechanisms such as neural networks and attention (Transformer).

  4. When the model generates a response, it predicts the next token in the sequence and continues until it reaches a defined length.

  5. Finally, the token sequence is converted back into human-readable text through a reverse tokenization process.

Tiktokenizer lets you explore the tokenization process by entering text and watching how it gets segmented into tokens. This is especially useful when you need to calculate the total number of tokens generated — something LLMs can't yet do on their own.

Despite their impressive capabilities, LLMs are not perfect. They can make mistakes — including inventing responses that seem plausible but are actually false, a phenomenon known as \"hallucination.\" Additionally, because they are trained on human-generated data, they can reproduce biases present in that data, including stereotypes.

Another challenge involves their ability to handle limited context: if a question exceeds the maximum size of the context window, important information may be overlooked. To address these limitations, it is essential to continuously test, adjust, and monitor these models.

In short, LLMs work like ultra-fast, tireless assistants. They analyze words, predict responses word by word, and draw on an enormous knowledge base. However, they remain imperfect and require careful oversight.

Think of it as a digital Josiane — able to deliver accurate answers in an instant, but one who can also make mistakes or need a second look.

Understand How an LLM Works

LLMs are built on artificial neural networks inspired by the workings of the human brain. These networks are composed of layers of interconnected neurons. Each connection between neurons has an associated weight. These weights are adjusted during training to allow the network to learn to recognize patterns in data. The networks used for LLMs are particularly large and complex, with billions of parameters (the connection weights).

Take the sentence: \"The pothole on Route D42, reported by farmer Dupont, will be repaired on Friday.\"

The attention mechanism detects that:

  • \"pothole\" is linked to \"D42\" (location) and \"repaired\" (action).

  • \"Friday\" depends on \"reported\" (time elapsed since notification).

This contextual analysis enables precise responses such as: \"Work will begin on [date] according to the technical department.\"

Before the network can perform these operations, it must learn from large datasets.

Two key phases: Training and inference

  • Training: An intensive phase during which the model ingests terabytes of text. Over the course of weeks, it adjusts its parameters to predict the missing word in sentences.

Today, virtually the entire web is used! This represents, after filtering, approximately 44 terabytes of data. To get a sense of what that means, check out FineWeb, a large-scale web corpus developed by Hugging Face.

  • Inference: The day-to-day usage phase. When Josiane types a question, the LLM activates its fixed parameters to generate a response. It's like a pianist playing a memorized piece — fast, but without the ability to improvise anything new.

However, training these models is extremely resource-intensive in terms of computing power and data. Moreover, their knowledge is frozen at the time of their training, meaning they cannot access recent information without additional enrichment. Despite these limitations, LLMs possess remarkable emergent properties — such as answering questions, summarizing texts, and performing automatic translation.

Explore the Transformer Architecture

Most modern LLMs use an architecture called the Transformer — a key innovation that appeared in 2017 (source) and radically transformed the field of natural language processing (NLP).

Transformers are built around a central mechanism: attention. Unlike sequential approaches that process words one at a time, attention allows the model to examine an entire text sequence simultaneously. Each word can thus \"look at\" all the other words in a sentence to identify which ones are most relevant. This ability to evaluate all possible relationships between words significantly improves contextual understanding.

Transformers consist of two main parts:

  • The encoder: which processes the input text.

  • The decoder: which generates the output text.
    Models like those in the GPT family use only the decoder portion to produce text.

The attention mechanism works in three main steps:

  1. Computing queries, keys, and values: For each word in a sequence, the model creates three vector representations:

  • Q (Query): What the word is \"looking for\" in the sequence

  • K (Key): What identifies the word

  • V (Value): The actual information the word contains

Computing attention scores: For each position in the sequence, the model computes scores indicating how much each word should \"pay attention\" to the others:

  • It computes the dot product between the query (Q) of the current word and the keys (K) of all other words

  • These scores are normalized using a softmax operation to obtain attention weights

Weighted aggregation: The model creates a new representation for each word by combining the values (V) of all words, weighted by the attention scores computed above.

This approach allows the model to assign more importance to certain words depending on context. For example, in the sentence \"The man who bought a red car parked it,\" the attention mechanism allows the model to understand that \"it\" refers to \"car\" rather than \"man\" by capturing the relationships between these words.

In the full architecture, this attention mechanism is applied several times in parallel (multi-head attention) and across different layers, allowing the model to capture increasingly complex relationships between words.

Dive deep into how the Transformer architecture works with this article from the Financial Times. You can also look under the hood of nano-GPT: LLM Visualization

Transformers have transformed NLP for several key reasons:

  1. Parallelization and efficiency
    Unlike sequential models such as recurrent neural networks (RNNs) or LSTMs, Transformers process the entire sequence simultaneously. This enables efficient parallelization during training, drastically reducing computation time and making it possible to use very large datasets.

  2. Handling long-range dependencies
    The attention mechanism captures complex relationships between words that are far apart in a sentence. While sequential models struggle to retain information over long distances, Transformers directly evaluate the importance of each word relative to all others, improving contextual understanding.

  3. Scalability and improved performance
    This modular architecture has made it possible to develop increasingly large models (such as GPT, BERT, etc.), capable of handling a wide variety of tasks with remarkable accuracy — from automatic translation to text summarization. The ability to incorporate extended contexts and adapt to diverse challenges has paved the way for a new generation of NLP applications.

Adapt Your LLM to Your Use Case

To adapt a Large Language Model (LLM) to the specific needs of the Trifouillis-sur-Loire town hall, it's essential to explore several approaches to optimize both the relevance and efficiency of responses. Prompt engineering and fine-tuning are two classic techniques for customizing language models. However, their respective limitations — sometimes insufficient effectiveness for prompting, and high cost (as well as inflexibility) for fine-tuning — push us to consider a complementary alternative: RAG (Retrieval-Augmented Generation).

Design Effective Instructions with Prompt Engineering

Prompt engineering involves formulating and optimizing the text instructions given to the LLM. When well designed, these instructions produce relevant responses without modifying the model itself. This method is particularly useful in a public-service context, where citizen questions vary widely.

What is a prompt?

A prompt is an instruction or question given to the LLM. The quality of the prompt directly influences the relevance and accuracy of the response. For example:

  • General prompt: \"What documents are needed to get married?\"

  • Contextual and specific prompt: \"What documents are needed for a civil marriage in Trifouillis-sur-Loire?\"

Here are some key principles for an effective prompt:

1. Precision:

  • Use specific terms.

  • Example: \"List the required documents for a civil marriage in France.\"

2. Context:

  • Add details to ground the response in a local situation.

  • Example: \"For a civil marriage at a small French town hall like Trifouillis-sur-Loire, what documents are required?\"

3. Actionability:

  • Structure the prompt to encourage a clear response from the model.

  • Example: \"List the steps to follow to register a birth at the town hall.\"

4. Iteration:

  • Test different phrasings to find the most effective one.

Common techniques:

  • Zero-Shot Prompting: Get a response without providing any examples.

  • Few-Shot Prompting: Guide the model by providing a few examples.

  • Chain of Thought Prompting: Encourage step-by-step reasoning.

While this method is fast and requires no modification to the model, it has notable limitations — particularly when it comes to accessing the town hall's internal business and private information. The model simply cannot draw on that internal data, which limits its ability to deliver truly context-aware responses.

Adapt the LLM to Local Specifics with Fine-Tuning

Fine-tuning involves retraining a pre-trained LLM on specific data so it can respond more relevantly to particular needs. This can include administrative procedures, local information, or regulations specific to a town hall.

Why might fine-tuning be relevant for a town hall?

At first glance, we might think it's useful for 2 reasons:

  • Local specialization: General-purpose LLMs often lack data on local regulations.

  • Style and tone improvement: Fine-tuning can help customize the model to adopt a formal and courteous administrative style, as expected in a town hall context.

For example:

  • Before fine-tuning: \"What documents are needed to get married?\" (Generic response, sometimes inaccurate).

  • After fine-tuning: \"At Trifouillis-sur-Loire, you will need to provide a government-issued ID, a proof of address less than 3 months old, and a full copy of your birth certificate.\"

Although highly effective for certain use cases, fine-tuning has major drawbacks, including:

1. Cost and complexity:

  • Full fine-tuning (standard fine-tuning) requires significant computational and data resources. It may not be cost-effective for small town halls.

2. Lack of flexibility:

  • Once fine-tuned, the model becomes less general-purpose and may respond incorrectly to questions outside its training domain.

  • Example: After fine-tuning on Trifouillis-sur-Loire's local procedures, the model might fail to correctly answer questions about other regions.

3. Complex updates:

  • Administrative information changes regularly (for example, new forms, legislative changes). Updating a fine-tuned model requires restarting the entire training process, which is time-consuming and costly.

Understand the Value of RAG

Faced with the limitations of prompt engineering and fine-tuning, we opted for a hybrid approach based on RAG (Retrieval-Augmented Generation). This method combines the generative power of an LLM with real-time access to an external knowledge base to deliver enriched, relevant responses.

  • Retrieval: searching for relevant data

  • Augmented: adding that data as context to the prompt

  • Generation: using the augmented prompt with an LLM to generate content

Here are the key steps in how it works:

This workflow can be read across these 5 main steps:

  1. Corpus: Collect the text documents or information sources to be analyzed.

  2. Chunking: Split the documents into smaller units (chunks) for more efficient management and analysis.

  3. Vectorization: Convert the chunks into numerical vectors — mathematical representations of the content in a multidimensional space.

  4. Indexing: Organize the vectors in a vector database to enable fast search and retrieval of relevant information.

  5. Vector database query: Query the vector database to identify the most relevant documents for a given question or need.

Key Takeaways

  • LLMs, trained on enormous text corpora, can generate coherent responses and simulate human-like interactions.

  • Their knowledge is frozen at the time of pre-training, making them unable to incorporate information from after that date.

  • These models can lack relevance when faced with specific or local requests, limiting their usefulness in precise contexts.

  • Without adjustments, they risk producing general or incomplete responses — inadequate for tasks requiring high accuracy.

  • These limitations shape the need for solutions like RAG, which is covered in detail throughout this course.
